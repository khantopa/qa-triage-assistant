  2. Admin block executes → `adminLoginPage.open()` navigates to admin domain **before** `deleteAllCookies()` → member-domain cookie survives
  3. Test navigates back to member page → `isAfterLoginPcrEligible()` sees `pcr_hide_after_login=true` → AfterLoginPCR modal suppressed
  4. "Complete now and get a FREE Boost" button (rendered by AfterLoginPCR) never mounts → assertion fails

- **What misled the original investigation**: `227a0ca07c` (ModalManager migration, 2026-03-09) touched `AfterLoginPCR.tsx` and coincided with the sprint window. The element ID (`free_profile_boost_count_down_container`) appeared to be a rendering regression. But the real cause was the cookie preventing the modal from ever being eligible — the ModalManager was working correctly.
